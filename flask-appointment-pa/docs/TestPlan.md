# Piano di Test

## Obiettivi
Verificare che il sistema soddisfi i requisiti funzionali e non funzionali.

## Casi di Test
- TC-001: Prenotazione valida → conferma e notifica.
- TC-002: Prenotazione doppia → errore.
- TC-003: Modifica prenotazione → aggiornamento slot.
- TC-004: Cancellazione prenotazione → slot liberato.
